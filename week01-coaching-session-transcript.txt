COACH:
Good intuition, but the distinction is actually about something different — it's
not about CommonJS vs ESM. Both styles here are ES Module syntax. The difference
is between default exports and named exports:

- No curly braces → you're importing the module's default export. express exports
  a single main function as its default, so you grab it directly.

- Curly braces → you're importing a named export. The url module exports multiple
  things (like fileURLToPath, URL, parse, etc.), and the curly braces let you pick
  exactly which one(s) you want.

Think of it like this: a default export is the "main thing" a module offers,
while named exports are like items on a menu — you pick what you need by name.
